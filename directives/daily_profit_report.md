# Daily Profit Report — Directive SOP

**Layer 1 — Standard Operating Procedure**

## Objective

Run once daily at 6:00 PM AEST (8:00 AM UTC via GitHub Actions cron).
Pull yesterday's data from Shopify, Meta Ads, and Klaviyo.
Calculate estimated operational profit.
Append one row to the Google Sheet log.
Send a profit summary email to the owner via Gmail.

---

## Inputs Required

| Input | Source | Where stored |
|-------|--------|-------------|
| Shopify store URL | Shopify Admin | `.env` → `SHOPIFY_STORE_URL` |
| Shopify access token | Shopify Admin → Apps → Private apps | `.env` → `SHOPIFY_ACCESS_TOKEN` |
| Meta access token | Meta Business Suite → System Users | `.env` → `FACEBOOK_ACCESS_TOKEN` |
| Meta ad account ID | Meta Ads Manager URL (act_XXXXXXX) | `.env` → `FACEBOOK_AD_ACCOUNT_ID` |
| Klaviyo API key | Klaviyo → Account → API Keys | `.env` → `KLAVIYO_API_KEY` |
| Google credentials | Google Cloud Console → OAuth 2.0 | `credentials.json` (project root) |
| Google token | Generated by `google_sheets_setup.py` | `token.json` (project root) |
| Spreadsheet ID | Printed by `google_sheets_setup.py` | `.env` → `SPREADSHEET_ID` |
| Gmail address | Your Gmail | `.env` → `SMTP_USER` + `REPORT_EMAIL` |
| Gmail App Password | [myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords) | `.env` → `SMTP_PASSWORD` |

## Scripts to Call (in order)

| Step | Script | Purpose |
|------|--------|---------|
| 1 | `execution/shopify_fetch_orders.py` | Fetch orders, revenue, refunds |
| 2 | `execution/meta_fetch_spend.py` | Fetch Meta ad spend |
| 3 | `execution/klaviyo_fetch_revenue.py` | Fetch Klaviyo email revenue |

| 5b | `execution/send_email_report.py` | Send Gmail report |

**Orchestrator (calls all of the above):**
```bash
python execution/run_daily_report.py
```

## Profit Formula

```
Total Revenue   = Net Revenue (ex tax, ex refunds) + Klaviyo Email Revenue
COGS cost       = COGS_PER_ORDER × orders
Shipping cost   = SHIPPING_PER_ORDER × orders
Payment fees    = (PAYMENT_FEE_PCT × net_revenue) + (PAYMENT_FEE_FIXED × orders)

Profit          = Total Revenue – COGS – Shipping – Payment Fees – Meta Ad Spend
Margin %        = Profit / Total Revenue × 100
```

Default constants (footwear, AUD — override in `.env`):

| Constant | Default | Notes |
|----------|---------|-------|
| `COGS_PER_ORDER` | `42.00` | Avg cost of goods per pair |
| `SHIPPING_PER_ORDER` | `9.50` | Avg AusPost/StarTrack fulfilment |
| `PAYMENT_FEE_PCT` | `0.0175` | Shopify Payments AU = 1.75% |
| `PAYMENT_FEE_FIXED` | `0.30` | $0.30 per transaction |

## One-Time Setup Checklist

Before the automation runs for the first time:

### Google Cloud Setup
- [ ] Go to [console.cloud.google.com](https://console.cloud.google.com)
- [ ] Create a project (or use existing)
- [ ] Enable **Google Sheets API** and **Google Drive API**
- [ ] Create OAuth 2.0 credentials (Desktop App type)
- [ ] Download `credentials.json` → place in project root
- [ ] Run `google_sheets_setup.py` locally (opens browser for auth)
- [ ] Copy the printed `SPREADSHEET_ID` into `.env`

### Gmail App Password
- [ ] Enable 2FA on your Google account (required for App Passwords)
- [ ] Go to [myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords)
- [ ] Create an App Password for "Mail" → "Windows Computer"
- [ ] Copy the 16-character password into `.env` → `SMTP_PASSWORD`

### GitHub Actions Secrets
In your GitHub repo → **Settings → Secrets and variables → Actions → New repository secret**:

| Secret Name | Value |
|-------------|-------|
| `SHOPIFY_STORE_URL` | Your store URL |
| `SHOPIFY_ACCESS_TOKEN` | Shopify admin token |
| `FACEBOOK_ACCESS_TOKEN` | Meta access token |
| `FACEBOOK_AD_ACCOUNT_ID` | `act_XXXXXXXXXX` |
| `KLAVIYO_API_KEY` | Klaviyo private key |
| `SPREADSHEET_ID` | From `google_sheets_setup.py` output |
| `SMTP_USER` | `you@gmail.com` |
| `SMTP_PASSWORD` | 16-char App Password |
| `REPORT_EMAIL` | `you@gmail.com` (same as above) |
| `GOOGLE_TOKEN_JSON` | Contents of `token.json` (entire file as text) |
| `COGS_PER_ORDER` | `42.00` (or your actual value) |
| `SHIPPING_PER_ORDER` | `9.50` (or your actual value) |
| `PAYMENT_FEE_PCT` | `0.0175` |
| `PAYMENT_FEE_FIXED` | `0.30` |

## Scheduling

GitHub Actions cron: `0 8 * * *` (8:00 AM UTC = 6:00 PM AEST / 7:00 PM AEDT)

> ⚠ **DST Note:** During Australian Daylight Saving Time (Oct–Apr, UTC+11), the email arrives at 7 PM AEDT instead of 6 PM. GitHub Actions does not support timezone-aware cron. Consider Render Cron or Railway for DST-precise scheduling.

## Edge Cases & Known Issues

| Scenario | Behaviour |
|----------|-----------|
| Zero orders yesterday | Shopify returns empty list → profit is negative (just ad spend + fixed costs). Normal. |
| No Meta campaigns running | `meta_fetch_spend.py` returns `ad_spend: 0.0` without error |
| No Klaviyo emails sent | `klaviyo_fetch_revenue.py` returns `email_revenue: 0.0` without error |
| Google Sheets write fails | Non-fatal — email is still sent, error logged to GitHub Actions output |
| Token.json expires | Re-run `google_sheets_setup.py` locally to refresh. Update `GOOGLE_TOKEN_JSON` secret. |
| Meta token expires | Meta user tokens expire after 60 days. Use a System User token (non-expiring) instead. |
| Shopify API version | Currently using `2024-01`. Check [Shopify changelog](https://shopify.dev/changelog) quarterly. |
